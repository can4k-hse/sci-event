# Create PR

Создай коммит и PR для текущей ветки по следующему алгоритму:

0. Если имя ветки передано аргументом (`$ARGUMENTS`) — используй его. Если аргумент не указан — спроси у пользователя название ветки перед тем как продолжать.
1. Если есть незакоммиченные изменения — выполни `git stash`.
2. Переключись на `main`: `git checkout main && git pull origin main`.
3. Переключись обратно на нужную ветку: `git checkout {ветка}`.
4. Если делали stash — восстанови: `git stash pop`.
5. Посмотри staged и unstaged изменения через `git diff HEAD` и `git status`.
6. Посмотри коммиты ветки относительно main через `git log main..HEAD --oneline`.
7. Сгенерируй краткое описание изменений на русском языке (1–2 предложения) — это будет {описание}.
8. Если есть что коммитить — создай коммит с сообщением в формате: `{ветка}: {описание}`
   - Пример: `FRONTFRAMEWORK-4: добавлены цветовые токены и CSS Modules для компонента Button`
9. Сделай rebase на main: `git rebase origin/main`. Если возникли конфликты — сообщи пользователю и остановись.
10. Запушь ветку в remote: `git push --force-with-lease -u origin {ветка}`.
11. Из вывода `git push` возьми ссылку вида `https://github.com/.../pull/new/{ветка}` и верни её пользователю. Не вызывай `gh` и не создавай PR программно.
